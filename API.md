# ESP-NOW Gateway Transmitter Serial API Reference

This document describes the serial protocol used for bidirectional communication between the ESP-NOW Gateway Transmitter and the MQTT Gateway module over the UART2 link (115200 baud).

All messages in both directions are JSON objects terminated by a newline (`\n`) character.

---

## 1. Gateway → Transmitter (Incoming Messages)

These are messages sent by the MQTT Gateway to control the transmitter or send packets over ESP-NOW.

### Send ESP-NOW Packet
To transmit a payload to an ESP-NOW peer device.
*   **JSON Schema**:
    ```json
    {
      "to": "<12-char hex MAC address, no colons>",
      "message": <arbitrary JSON object payload>
    }
    ```
*   **Example**:
    ```json
    {
      "to": "ECFABC2FE867",
      "message": {
        "channel": 1,
        "push": 500
      }
    }
    ```

### Control Commands
Special management commands executed by the transmitter.

#### Ping
Check if the transmitter is responsive and query system status.
*   **Request**:
    ```json
    {"command": "ping"}
    ```

#### Reset / Reboot
Force the transmitter to reboot.
*   **Request**:
    ```json
    {"command": "reset"}
    ```

#### Get MAC Address
Query the active MAC address of the transmitter.
*   **Request**:
    ```json
    {"command": "get-mac"}
    ```

#### Set Custom MAC Address
Configure a custom MAC address (saved to NVS, takes effect after automatic reboot).
*   **Request**:
    ```json
    {"command": "set-mac", "value": "AABBCCDDEEFF"}
    ```

---

## 2. Transmitter → Gateway (Outgoing Messages)

These are JSON messages sent by the transmitter to the gateway. Each message has a `"type"` key.

### Data Message (`type`: "data")
Emitted when the transmitter successfully receives and decrypts (if enabled) an ESP-NOW packet from a peer.
*   **Fields**:
    *   `type`: Always `"data"`.
    *   `mac`: MAC address of the source peer (12 hex characters).
    *   `message`: The JSON payload sent by the peer.
*   **Example**:
    ```json
    {
      "type": "data",
      "mac": "ECFABC2FE867",
      "message": {
        "status": "ok"
      }
    }
    ```

### Log Message (`type`: "log")
System and debugging logs generated by the transmitter.
*   **Fields**:
    *   `type`: Always `"log"`.
    *   `from`: The source component name or peer MAC address.
        *   Transmitter component logs: `WHO_AM_I` (default: `"ESP32-ESPNOW-GW-TRANS"`).
        *   Peer-specific logs (e.g. delivery fails): Peer's MAC address (e.g., `"ECFABC2FE867"`).
    *   `level`: Log level (`"info"`, `"warning"`, `"error"`, `"debug"`).
    *   `message`: The text description of the log event.
*   **Example**:
    ```json
    {
      "type": "log",
      "from": "ESP32-ESPNOW-GW-TRANS",
      "level": "info",
      "message": "Starting ESP-NOW Gateway Transmitter..."
    }
    ```

### Heartbeat Message (`type`: "heartbeat")
Emitted periodically by the transmitter at a configured interval (`HEART_BEAT_S` in `config.h`) to notify the gateway of system health.
*   **Fields**:
    *   `type`: Always `"heartbeat"`.
    *   `device`: Device identifier string (e.g. `"ESP32-ESPNOW-GW-TRANS"`).
    *   `mac`: Current MAC address of the transmitter.
    *   `uptime`: System uptime in seconds.
    *   `peers`: Count of registered ESP-NOW peers.
    *   `free_heap`: Free heap memory in bytes.
*   **Example**:
    ```json
    {
      "type": "heartbeat",
      "device": "ESP32-ESPNOW-GW-TRANS",
      "mac": "30AEA4070D64",
      "uptime": 3600,
      "peers": 2,
      "free_heap": 218440
    }
    ```

### Command Responses (`type`: "response")
Returned immediately after receiving control commands from the gateway.

#### Ping Response
*   **Example**:
    ```json
    {
      "type": "response",
      "command": "ping",
      "status": "success",
      "device": "ESP32-ESPNOW-GW-TRANS",
      "mac": "30AEA4070D64",
      "uptime": 1234,
      "peers": 2,
      "free_heap": 218440
    }
    ```

#### Get MAC Response
*   **Example**:
    ```json
    {
      "type": "response",
      "command": "get-mac",
      "status": "success",
      "mac": "30AEA4070D64"
    }
    ```

#### Set MAC Response
*   **Example**:
    ```json
    {
      "type": "response",
      "command": "set-mac",
      "status": "success",
      "mac": "AABBCCDDEEFF",
      "message": "Rebooting to apply new MAC address"
    }
    ```

#### Reset Response
*   **Example**:
    ```json
    {
      "type": "response",
      "command": "reset",
      "status": "success",
      "message": "Rebooting device..."
    }
    ```

#### Error Response (e.g. Invalid command or parameters)
*   **Example**:
    ```json
    {
      "type": "response",
      "command": "unknown",
      "status": "error",
      "message": "Unknown command: invalid_cmd"
    }
    ```
